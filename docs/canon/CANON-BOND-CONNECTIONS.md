## CODE NAMING CONVENTION

### Suffix system

For planetary codes (Moon through Pluto + Cosmic Reward + Cosmic Lesson):

| Suffix | Meaning |
|---|---|
| F | First (their card sits in your position, Life Spread) |
| FS | First Spiritual (their card sits in your position, Spiritual Spread) |
| R | Reverse (you sit in their position, Life Spread) |
| RS | Reverse Spiritual (you sit in their position, Spiritual Spread) |
| M | Mutual (both sit in each other's positions, Life Spread) |
| MS | Mutual Spiritual (both sit in each other's positions, Spiritual Spread) |

### Why some planets have only 4 codes (Neptune, Pluto, Cosmic Reward, Cosmic Lesson)
Mutual positions for these outer/cosmic positions are mathematically rare or structurally not present in the Cardology system. The First/Reverse + Spiritual variants are sufficient.

### Why some have 6 codes (Moon, Venus, Mars, Jupiter, Saturn, Uranus)
These are personal planets where Mutual connections occur with reasonable frequency.

## SATURN SAFETY FOOTER PROTOCOL

When ANY Saturn code (SAF, SAFS, SAR, SARS, SAM, SAMS) appears in the connection set for a Bond report, the system AUTO-ATTACHES the safety footer.

### Safety footer text

```
This report explores patterns in connection. It is not a diagnostic
or therapeutic tool. If you're experiencing fear, control, or harm
in this relationship, please reach out for support.

UK National Domestic Violence Helpline: 0808 2000 247 (24 hours, free)
Refuge: refuge.org.uk
The Samaritans: 116 123

Your safety comes first. Always.
```

### Where the footer appears
- Bottom of the Bond report (last page in PDF, last section in portal)
- Visible BEFORE the closing letter so it isn't buried
- In a clearly bordered box (gold border, magenta accent on the helpline number)

### Logic in admin Bond Builder
- Connection dropdowns 1-5 each have a flag: `triggers_safety_footer: BOOLEAN`
- All 6 Saturn codes have this flag set to TRUE
- When admin saves the Bond report, the system checks all 5 dropdown selections
- If ANY dropdown contains a code with `triggers_safety_footer: TRUE`, the footer attaches automatically
- Yemi sees a confirmation: "Saturn code present — safety footer attached"
- Cannot be disabled (compliance rule)

### Additional flagging (for admin awareness)
- KRMA + Saturn = extra QA pass before delivery
- POWR in romantic context = extra QA pass before delivery
- 3+ Pluto codes in same connection set = extra QA pass

These flags don't change the report. They surface in the admin builder for Yemi to do an extra read-through.
